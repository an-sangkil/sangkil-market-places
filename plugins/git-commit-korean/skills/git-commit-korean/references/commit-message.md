# 한글 커밋 메시지 가이드

## 제목 형식
`<유형>: <변경 요약>`

유형 예시:
- `feat`: 기능 추가
- `fix`: 버그 수정
- `refactor`: 동작 변경 없는 구조 개선
- `test`: 테스트 추가/수정
- `docs`: 문서 수정
- `chore`: 빌드/설정/의존성 변경

## 제목 작성 규칙
- 한글로 작성한다.
- 한 문장으로 핵심을 드러낸다.
- 모호한 표현("수정", "변경")만 단독으로 쓰지 않는다.

## 본문 작성 규칙
- 필요할 때만 작성한다.
- 무엇을 바꿨는지보다 왜 바꿨는지 우선 설명한다.
- 여러 변경점은 항목으로 분리한다.

## 예시
- `feat: 사용자 프로필 이미지 업로드 기능 추가`
- `fix: 로그인 세션 만료 시 리다이렉트 누락 수정`
- `refactor: 결제 모듈 예외 처리 분기 정리`
- `test: 주문 생성 API 유효성 검증 케이스 추가`
