# document/

프로젝트 문서 루트 디렉토리.

## 구조

| 폴더 | 용도 | 작성 시점 |
|------|------|----------|
| `architecture/` | 공통 아키텍처 규칙 (레이어, 패키지 컨벤션 등) | 프로젝트 초기 |
| `exec-plans/` | 기능별 요구사항, 실행계획, TODO | `/spec`, `/plan` 커맨드 |
| `api-spec/` | API 엔드포인트 명세 | 구현 완료 후 |
| `product-specs/` | 제품 스펙 (파싱 규칙, DB 스키마, 비즈니스 로직 등) | 수시 |

## 규칙

- `architecture/` 파일은 모든 기능 문서의 상위 참조 — 기능 문서에 공통 구조를 중복 서술하지 않는다.
- `exec-plans/{domain}/` 하위에 기능별 문서를 생성한다.
- `api-spec/{domain}/` 하위에 엔드포인트별 명세를 생성한다.
- 구현으로 동작이 변경되면 같은 작업 사이클에서 관련 문서도 함께 갱신한다.
