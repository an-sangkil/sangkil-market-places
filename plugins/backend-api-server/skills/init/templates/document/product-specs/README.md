# docs/product-specs/

제품 스펙 문서를 관리하는 디렉토리.

## 파일 구조

```
product-specs/
├── {feature}/
│   ├── overview.md           ← 기능 개요, 비즈니스 목적
│   ├── parsing-rules.md      ← 데이터 파싱/변환 규칙
│   ├── db-schema.md          ← 테이블 구조, 인덱스 전략
│   └── ...
```

## 작성 규칙

- 기능(feature) 단위로 하위 폴더를 생성한다.
- 구현 전 참조하는 **기획/스펙 문서** — 에이전트가 구현 시 이 문서를 근거로 코드를 작성한다.
- 구현 결과가 스펙과 달라지면 스펙도 함께 갱신한다.
