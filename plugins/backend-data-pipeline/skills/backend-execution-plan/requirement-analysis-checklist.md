# Requirement Analysis Checklist

## 1) 비즈니스 목표 확인
- 해결하려는 문제를 한 문장으로 설명했는가
- 성공 기준(KPI/지표)을 명시했는가
- 배포 목표일과 출시 범위를 명시했는가

## 2) 기능 요구사항 확인
- 사용자 액터가 명확한가
- 주요 사용자 시나리오가 단계별로 정리되었는가
- 입력/출력/예외가 정의되었는가
- 완료 기준(acceptance criteria)이 있는가

## 3) 비기능 요구사항 확인
- 성능 목표(RPS, 지연 시간, 처리량)가 있는가
- 가용성/SLA 요구가 있는가
- 보안/개인정보 요구가 있는가
- 감사 로그/추적성 요구가 있는가

## 4) 도메인/데이터 확인
- 핵심 엔티티와 관계가 정의되었는가
- 데이터 소스와 진실 원천(source of truth)이 정의되었는가
- 데이터 정합성 규칙이 있는가
- 보관/파기 정책이 있는가

## 5) 연동/인터페이스 확인
- 외부 API/시스템 의존성이 식별되었는가
- 요청/응답 계약이 충분히 정의되었는가
- 장애 시 재시도/타임아웃/서킷브레이커 정책이 있는가
- 이벤트 발행/구독 규칙과 순서 보장 정책이 있는가

## 6) 운영 요구사항 확인
- 모니터링 지표/알람 기준이 있는가
- 배치/스케줄 작업의 실패 복구 전략이 있는가
- 롤백 또는 기능 플래그 전략이 있는가
- 운영 주체와 온콜 경로가 있는가

## 7) 전달/협업 확인
- 의사결정 필요 항목과 담당자가 명확한가
- 미정 항목의 마감일이 설정되었는가
- 팀 간 핸드오프 산출물이 명시되었는가

## 8) 최종 판단
- 즉시 개발 착수가 가능한가
- 불확실성이 높은 항목은 무엇인가
- 스프린트에 넣을 최소 실행 범위(MVP)는 무엇인가
