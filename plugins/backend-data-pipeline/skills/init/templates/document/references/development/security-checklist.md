# Security Checklist 레퍼런스

이 프로젝트의 **보안 체크리스트**이다. 데이터 수집/처리 파이프라인은 외부 데이터를 다루므로 **입력 검증과 민감 정보 보호**에 높은 기준을 적용한다.

`development` 에이전트는 구현 시, `test-qa` 에이전트는 QA 검증 시, `deploy-prep` 에이전트는 배포 체크 시 이 문서를 참조한다.

> **NOTE:** 프로젝트의 도메인 특수성(의료 데이터, 금융 데이터 등)에 따라 이 문서를 커스터마이즈한다.

---

## 1. SQL Injection 방지 (MyBatis)

### 규칙
- 파라미터 바인딩은 **반드시 `#{}` 사용**
- `${}`는 **원칙적으로 금지** (문자열 치환, Injection 위험)
- `${}`가 필요한 경우(테이블명 동적 지정 등)는 **화이트리스트 검증 필수**

### 체크 항목
- [ ] 신규 Mapper XML의 모든 파라미터가 `#{}` 사용
- [ ] `${}`가 있다면 테이블명/컬럼명 등 불가피한 경우인가
- [ ] `${}` 사용처에 화이트리스트 검증이 있는가

---

## 2. 민감 정보 로깅 방지

### 금지: 다음 정보를 로그에 남기지 않는다
- 개인식별정보 (PII): 이름, 전화번호, 이메일, 주민등록번호
- 도메인 특화 민감정보 (PHI/재무정보 등): 프로젝트에 따라 정의
- 원시 데이터 전체 (Bundle, JSON payload 등)
- 인증 토큰, API 키, 비밀번호

### 허용: 식별자만 로깅
- 내부 식별자 (`userId`, `jobId`, `reportId` 등)
- HTTP 상태 코드, 응답 시간, 에러 타입

### 체크 항목
- [ ] 신규 로그 문에 원시 데이터 전체를 찍는 코드가 없는가
- [ ] 민감 정보 값이 로그 메시지의 `{}`에 직접 들어가지 않는가
- [ ] 예외 메시지에 민감 정보가 포함되지 않는가

---

## 3. 입력 값 검증

### 검증 지점
- **시스템 경계**: 메시지 큐 수신, 외부 API 응답, 사용자 입력
- **내부 컴포넌트 간**: 필요 없음 (내부 신뢰)

### 체크 항목
- [ ] 메시지 수신 지점에 검증 로직이 있는가
- [ ] 외부 API 응답에 예상치 못한 값이 들어와도 안전한가
- [ ] 검증 실패 시 명확한 예외 타입을 던지는가

---

## 4. 암호화

### 규칙
- 설정 파일에 평문 시크릿 금지 (Jasypt `ENC(...)` 등 사용)
- 암호화 키는 **환경변수**로 주입
- Git에 평문 시크릿 커밋 금지

### 체크 항목
- [ ] 신규 설정 속성 중 시크릿 성격은 암호화되었는가
- [ ] 설정 파일에 평문 시크릿이 없는가
- [ ] 암호화 키가 리포에 커밋되지 않는가

---

## 5. 인증 / 인가

### 체크 항목
- [ ] 수집 요청 처리 전 권한 확인이 되는가
- [ ] 권한 없는 접근 시 명확한 예외 처리
- [ ] 토큰 만료 시 재발급 로직이 있는가

---

## 6. 외부 API 호출

### 규칙
- 외부로 보낼 데이터는 **최소화 원칙**
- HTTPS 강제 사용 (HTTP 금지)
- 인증서 검증 끄지 않기 (`trustAll` 금지)

### 체크 항목
- [ ] 외부 URL이 `https://`로 시작하는가
- [ ] 인증서 검증을 우회하는 코드가 없는가
- [ ] 외부로 보내는 페이로드에 불필요한 민감 정보가 포함되지 않는가

---

## 7. 데이터 파싱 특화 보안

### 규칙
- 파싱 중 예외 발생 시 스택 트레이스에 원시 데이터가 포함되지 않도록 주의
- 식별자를 로그에 남기는 건 허용, 원시 데이터 내용은 금지
- 파싱 후 사용하지 않는 필드는 즉시 버림 (메모리 체류 최소화)

### 체크 항목
- [ ] 파싱 예외 처리 시 원시 데이터 전체를 재throw하지 않는가
- [ ] 파싱 결과 객체의 toString()이 민감 정보를 노출하지 않는가
- [ ] Lombok `@ToString(exclude=...)`로 민감 필드가 제외되어 있는가

---

## 8. 심각도 분류 (QA 리포트용)

| 심각도 | 예시 |
|---|---|
| **Critical** | SQL Injection 취약점, 인증 우회, 평문 시크릿 커밋, 민감 정보 평문 로깅 |
| **Major** | 입력 검증 누락, 인증서 검증 끄기, 민감 정보 로깅 가능성 |
| **Minor** | 로그 포맷 개선 여지, `@ToString` 필드 제외 권장 |
