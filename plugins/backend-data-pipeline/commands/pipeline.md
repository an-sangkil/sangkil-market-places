---

사용자가 제공한 요구사항으로 전체 파이프라인을 실행한다.

**요청 내용:**
$ARGUMENTS

**실행 방식:**
- Agent tool로 `pipeline` 에이전트를 호출한다
- 위 요청 내용을 그대로 전달한다
- 파이프라인 에이전트의 단계별 게이트 규칙을 따른다 (1/2/5/6/7단계는 사용자 승인, 3/4단계는 자동 루프)
- 각 단계 산출물은 `docs/design/{feature-name}/` (설계 문서) 및 `docs/output/{feature-name}/` (결과물) 아래에 저장된다

**추가 정보가 필요한 경우:**
사용자에게 부족한 정보(Figma URL, Notion URL, API 명세 등)를 물어본 뒤 진행한다.
